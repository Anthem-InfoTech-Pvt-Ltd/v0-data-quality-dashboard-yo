# Database Seeding Instructions

## Prerequisites

1. Make sure MongoDB is running (local or Atlas)
2. Create a `.env.local` file in the root directory with your MongoDB connection string

## Setup

1. **Create `.env.local` file:**

```env
MONGODB_URI=mongodb://localhost:27017/dataqualitydashboard
```

For MongoDB Atlas:
```env
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/dataqualitydashboard
```

2. **Run the seed script:**

```bash
npm run seed
```

## What the Seed Script Does

The seed script will:
- ✅ Connect to your MongoDB database
- ✅ Clear existing collections (companies, contacts, alerts, emailAlerts, leads)
- ✅ Insert demo data with **unique MongoDB ObjectIds** for each document
- ✅ Display a summary of inserted records

## Troubleshooting

### Error: MONGODB_URI is not set
- Make sure you have created `.env.local` file in the root directory
- Check that the file contains `MONGODB_URI=your_connection_string`

### Error: Connection failed
- Verify MongoDB is running
- Check your connection string is correct
- For Atlas, make sure your IP is whitelisted

### Error: Cannot find module 'dotenv'
- Run: `npm install`

## Notes

- All documents automatically receive unique MongoDB ObjectIds (no need to specify them)
- The `_id` field is automatically generated by MongoDB
- You can run the seed script multiple times - it will clear and re-seed the data
